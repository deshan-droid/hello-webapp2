üöÄ CI/CD Pipeline Flow

A practical pipeline that matches real-world DevOps: from code commit to safe deployment with verification.

1) Source
Developers push code to Git. Jenkins checks out the repo using webhook or polling.
2) Build
Maven compiles & packages the app into a JAR (or WAR). Artifacts are produced per commit.
3) Test
Automated tests run. If tests fail, pipeline stops and deployment is blocked.
4) Deploy
Jenkins connects via SSH to the server, uploads the JAR and starts the service.
5) Verify
Health check hits /actuator/health to confirm service is running properly.
6) Switch
Nginx routes traffic to the new port. If anything breaks, rollback to the previous port.

Blue/Green Deployment Strategy

Blue/Green
Run two versions side-by-side: Blue (live) and Green (new). Deploy to standby, verify, then switch traffic.
Zero-downtime releases
Fast rollback
Safer deployments
Health Checks
After starting the service, the pipeline verifies it using an endpoint like: /actuator/health
Prevents broken releases
Confirms real readiness
Automates rollback trigger

pipeline {
  agent any
  stages {
    stage('Checkout') { steps { git 'https://github.com/your-repo.git' } }
    stage('Build')    { steps { sh 'mvn -B clean package' } }
    stage('Test')     { steps { sh 'mvn -B test' } }
    stage('Deploy')   { steps { sh './deploy-blue-green.sh' } }
    stage('Verify')   { steps { sh 'curl -f http://server:8087/actuator/health' } }
  }
}
